Billing

How billing & credits work

Everything works exactly like official Kiro — same billing cycle, same credits, same features. We just provide the subscription at 50% off.

Fractional credit system

Credits are consumed fractionally based on each request. Simple edits use fewer credits than complex tasks. Charged in 0.01 credit increments, so simple prompts can cost less than 1 credit.

Monthly renewal

Credits reset at the start of each billing cycle. Unused credits don't roll over. Your subscription renews monthly — cancel anytime, no lock-in.

Overages (opt-in)

On paid tiers, you can opt-in to pay-as-you-go overages at $0.04/credit if you exceed your monthly limit. This is optional — you won't be charged unexpectedly.

Upgrades & downgrades

Upgrade anytime — you get immediate access to the new tier's full credits (prorated billing). Downgrades take effect next billing cycle.

What counts as a credit?

Vibe requests

  • Chatting with Kiro ("explain this code")
  • Creating or refining spec documents
  • Triggering agent hooks
  • Complex prompts ("build an API") may use multiple

Spec requests

  • Executing a task from tasks.md = 1 spec request
  • Running multiple tasks ("do tasks 1-3") = 3 spec requests
  • Subtasks each count as 1 spec request
  • Specs mode is more efficient for complex projects

Everything above works identically for our subscriptions. We provide the same Kiro account — the only difference is you pay 50% less.
